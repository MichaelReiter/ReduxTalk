Topics Covered

What Is Redux?

  • Redux is a predictable state container for JavaScript apps
  • Redux implements Flux

What is Flux?

Summary

  • Whenever a user interacts with a view, an action is dispatched
  • Stores containing the application's data update the affected views

Redux simplifies Flux

  • Components are passed callback functions as props
  • Callbacks create and dispatch actions whenever a UI event is triggered
  • Actions are processed, computing the updated state
  • State is kept immutable
  • State is saved in a single store serving the entire application
  • Components are rerendered as necessary, receiving updated state as props

Why Redux?

Unidirectional Data Flow

  • Easy to reason about application state

Powerful State Tree

  • Immutable state and pure reducers allow trivial time travel

Hot Reloading

  • Logic can be updated on the fly without affecting state

Concepts

Actions

  • Objects describing an intended state update
  • Must contain a type field describing the update
  • Can otherwise take any shape
  • The only way store changes can be triggered
  • Should be dispatched whenever a user interaction occurs

Reducers

  • Process actions
  • Pure functions (no side effects)
  • (state, action) => new state

Store

  • Large nested object for keeping track of application state
  • Only one store in Redux
  • Root reducer composed of smaller reducers which can independently manipulate state
  • Similar to composing React components
